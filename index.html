1) Where art thou

一开始看到题目，笔者以为是FCC中文站翻译出错，应该是where are you，后来查了一下thou的意思，居然有”你”的意思。

题目：

code1：

function where(collection, source) {
      var arr = [];
     // What's in a name?
     var sourArr = Object.keys(source);
     for(var i =0, len1 = collection.length; i < len1; i++) {
           var num = 0;
        for(var j = 0,len2 = sourArr.length; j < len2; j++) {
            if(!collection[i].hasOwnProperty(sourArr[j])){
                break;
             } else {
                    if(collection[i][sourArr[j]] === source[sourArr[j]]) {
                      num += 1;
                }
              }
            }
           if( num ===len2 ) {
              arr.push(collection[i]);
        }
     }
     return arr;
}

//test
where([{ "a": 1, "b": 2 }, { "a": 1 }, { "a": 1, "b": 2, "c": 2 }], { "a": 1, "b": 2 });
  • 可以看到上述代码用了“无数”个for循环和if-else判断，缺少注释。

code2:

function where(collection, source) {
      var keys = Object.keys(source);   //获得source的key数组
      return collection.filter(function(obj){  //筛选出使keys满足断言函数的对象元素
        return  keys.every(function(e){ //断言函数判断obj是否包含所有source的键值对
              return obj.hasOwnProperty(e) && obj[e] === source[e];
        });
      });
}

//test
where([{ "a": 1, "b": 2 }, { "a": 1 }, { "a": 1, "b": 2, "c": 2 }], { "a": 1, "b": 2 });
  • code2 明显精简了许多，增加了必要的注释。
  • 采用逆向思维解此题：collection数组中筛选符合一定条件的对象元素返回 <= 设计合适的断言函数 <= 采用逻辑与表达式来对每个键值对进行判断。

2)Pig Latin

题目：

code1：

function translate(str) {
      var letters = str.split("");
      var newStr = "";
      for(var i = 0, len = letters.length; i < len; i++) {
           if("aeiou".indexOf(letters[i]) !== -1) {
            var index = i;
              if (index === 0) {
            letters.push("way");
            newStr = letters.join("");       
         } else {
            var change = str.substr(0,index);
                newStr = str.substr(index) + change + "ay";
        }
        break;
         }    
      }
    return newStr;
}

//test
translate("glove");

code2:

function translate(str) {
      var newStr = "";
      var index = str.search(/[aoeiu]/);  //获得第一个元音字母的索引值，并根据索引值是否为0来判断返回什么样的值：
      return index === 0 ? str + "way" : str.substr(index) + str.substr(0,index) +"ay";
}

//test
translate("paragraphs");
  • code2 完爆 code1 ！

如何使用在线图床

一、工具选择

  • 建站工具: hexo.io

  • 虚拟主机: github.com

  • 域名注册: west.cn

  • 云服务商(图片存储): qiniu.com

  • Markdown编辑器： Atom

二、使用指南

1.使用Hexo + GithubPages 新建一个静态博客，并绑定域名。参考链接：点击

2.申请七牛云账号，随后开通文件存储空间(公开)，目前可以免费获得一个测试域名。如下图所示(若看不清细节，请右键单独打开图片)：

3.在博客所在github repo中新建文件夹“qiniuimgbed”用于存放文章相关图片，为方便管理，图片按 “20161008XXX_XXX.jpg” 形式命名，其中 “_“ 前为前缀命名，包含文章发布时间和文章名称(简写，并与其它图片一致)，”_“后为图片简称(与同一文章的其它图片区分)。如下图所示：

4、在七牛云空间中设置镜像源地址。按下图所示操作即可：

5、每次写文章时按下列步骤操作：

1). 写好文本

2). 根据内容添加图片或截图至本地”qiniuimgbed”文件夹，同时按规则命名图片文件

3). 在MD文档中添加 七牛云前缀地址+图片名的图片URL。如：

http://ocbao1wc2.bkt.clouddn.com/20161008tuchuang_jiangxiang2.jpg

4). deploy文章至github服务器即可。

三、 工作原理

  • 在网络中首次访问该文章时，客户端(浏览器等)会根据文档图片url访问七牛云获取图片资源;

  • 很明显七牛云中并没有相关图片，于是会根据镜像源地址访问随文档一起push至github中的图片，提供给客户端的同时会备份至相应绑定的存储空间;

  • 下一次再访问该文章时，则可从七牛云服务器中直接获取资源，从而提升图片资源的访问速度。
